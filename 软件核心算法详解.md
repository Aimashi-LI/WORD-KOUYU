### 1.2 单词级别的掌握率调整因子

**计算位置**：`/workspace/projects/client/algorithm/fsrs.ts`

#### 掌握率调整因子计算

**函数**：`calculateReviewTiming(word: Word)`

**理论基础**：
- 提前复习：记忆痕迹未充分巩固，过度学习导致记忆假象
- 延后复习：遗忘曲线下降，间隔效应失去作用
- 按时复习：必要难度理论，记忆效果最佳

**计算公式**：

```typescript
// 1. 提前复习（提前时间 > 3小时）
if (timeDiffHours < -3) {
  const earlyHours = Math.abs(timeDiffHours);
  // 非线性惩罚：提前时间越长，调整因子越小
  const penalty = Math.min(0.3, 0.07 * Math.log(earlyHours + 1));
  masteryAdjustmentFactor = 1.0 - penalty;  // 降低15%-30%
  reviewStatus = 'early';
}

// 2. 延后复习（延后时间 > 6小时）
if (timeDiffHours > 6) {
  const lateHours = timeDiffHours;
  // 指数惩罚：延后时间越长，调整因子越小
  // 延后6小时：factor ≈ 0.6
  // 延后12小时：factor ≈ 0.5
  // 延后24小时：factor ≈ 0.4
  const penalty = Math.min(0.6, 0.4 * (1 - Math.exp(-lateHours / 12)));
  masteryAdjustmentFactor = 1.0 - penalty;  // 降低20%-40%
  reviewStatus = 'late';
}

// 3. 按时复习（提前≤3小时且延后≤6小时）
masteryAdjustmentFactor = 1.0;  // 无惩罚
reviewStatus = 'on-time';
```

**掌握率调整因子应用**：

```typescript
// 应用掌握率调整因子到得分
const weightedScore = calculateWeightedScore(word, score, responseTime);
const adjustedScore = weightedScore * masteryAdjustmentFactor;

// 使用调整后的得分更新难度和稳定性
if (adjustedScore >= 4) {
  newDifficulty = Math.max(0, word.difficulty - 0.1); // 降低难度
} else if (adjustedScore <= 2) {
  newDifficulty = Math.min(1, word.difficulty + 0.1); // 增加难度
}

const newStability = predictNextInterval(
  { ...word, difficulty: newDifficulty },
  adjustedScore  // 使用调整后的得分计算稳定性
);
```

**调整因子范围**：

| 复习时机 | 时间范围 | 调整因子 | 稳定性影响 |
|---------|---------|---------|-----------|
| **提前复习** | >3小时前 | 0.7-0.85 | 降低15%-30% |
| **按时复习** | ±3-6小时 | 1.0 | 无影响 |
| **延后复习** | >6小时后 | 0.6-0.8 | 降低20%-40% |

### 1.3 加权得分计算

**计算位置**：`/workspace/projects/client/algorithm/fsrs.ts`

```typescript
function calculateWeightedScore(word: Word, score: number, responseTime: number): number {
  const avgTime = word.avg_response_time || TIME_CONFIG.BASE_TIME;
  const stdDev = avgTime * TIME_CONFIG.STD_DEV_RATIO;

  let weight = 1.0;

  // 快速响应（< 70% 平均时间）：提权，延长间隔
  if (responseTime < avgTime * TIME_CONFIG.FAST_RESPONSE_THRESHOLD) {
    weight = TIME_CONFIG.FAST_RESPONSE_WEIGHT;  // 1.2
  }

  // 慢速响应（> 130% 平均时间）：降权，缩短间隔
  if (responseTime > avgTime * TIME_CONFIG.SLOW_RESPONSE_THRESHOLD) {
    weight = TIME_CONFIG.SLOW_RESPONSE_WEIGHT;  // 0.8
  }

  const weightedScore = score * weight;
  return Math.min(6, Math.max(0, weightedScore));  // 限制在 0-6 分
}
```

**回忆时间权重配置**：

```typescript
export const TIME_CONFIG = {
  BASE_TIME: 20,                 // 基础时间（秒）
  DIFFICULTY_FACTOR: 10,         // 难度加成系数（秒）
  STD_DEV_RATIO: 0.3,            // 标准差比例
  FAST_RESPONSE_THRESHOLD: 0.7,  // 快速响应阈值（倍数）
  SLOW_RESPONSE_THRESHOLD: 1.3,  // 慢速响应阈值（倍数）
  FAST_RESPONSE_WEIGHT: 1.2,     // 快速响应权重
  SLOW_RESPONSE_WEIGHT: 0.8,     // 慢速响应权重
} as const;
```

```typescript
export const MASTERY_CONFIG = {
  // 稳定性阈值（天）- 超过此天数的单词很难遗忘
  // 0.5 天 = 约 12 小时，如果单词在 12 小时内能保持记忆，说明已掌握
  stabilityThreshold: 0.5,

  consecutiveHighScores: 2,

  // 高分标准 - 最低多少分才算高分
  highScoreThreshold: 5,
} as const;
```

### 2.2 双锁机制判断逻辑

**判断位置**：`/workspace/projects/client/screens/review-detail/index.tsx`

```typescript
const checkMasteryWithConfig = (word: Word, recentScores: number[]): boolean => {
  // 条件1：稳定性达到阈值
  const condition1 = word.stability >= MASTERY_CONFIG.stabilityThreshold;  // ≥0.5天

  // 条件2：最近N次得分都≥高分标准
  const condition2 = recentScores.length >= MASTERY_CONFIG.consecutiveHighScores &&
    recentScores.slice(0, MASTERY_CONFIG.consecutiveHighScores).every(s => s >= MASTERY_CONFIG.highScoreThreshold);  // 连续2次≥5分

  // 调试日志
  console.log(`[checkMasteryWithConfig] 单词: ${word.word}`);
  console.log(`[checkMasteryWithConfig]   稳定性: ${word.stability.toFixed(2)} 天 (阈值: ${MASTERY_CONFIG.stabilityThreshold} 天)`);
  console.log(`[checkMasteryWithConfig]   最近得分: [${recentScores.join(', ')}] (需要: ${MASTERY_CONFIG.consecutiveHighScores}次 ≥ ${MASTERY_CONFIG.highScoreThreshold}分)`);
  console.log(`[checkMasteryWithConfig]   条件1 (稳定性≥阈值): ${condition1}`);
  console.log(`[checkMasteryWithConfig]   条件2 (连续高分): ${condition2}`);
  console.log(`[checkMasteryWithConfig]   是否已掌握: ${condition1 && condition2}`);

  return condition1 && condition2;
};
```

### 2.3 双锁机制详解

#### 锁1：稳定性≥0.5天

**含义**：
- 稳定性表示单词在记忆中能保持的时间
- 0.5天 = 约12小时
- 如果单词在12小时内能保持记忆，说明已经形成了稳定的记忆痕迹

**为什么是0.5天？**
- 相比传统的14天标准，更适合频繁复习的场景
- 减少用户等待时间，提高学习体验
- 结合锁2（连续高分），确保掌握质量

#### 锁2：最近2次得分≥5分

**含义**：
- 需要连续两次高分（≥5分）才能标记为已掌握
- 确保记忆的稳定性和持续性
- 避免偶然高分的误判

**评分标准**：

```typescript
export const SCORING_CONFIG = {
  // 两种方式都正确
  PERFECT_SCORE: 6,      // 完美得分
  // 只有一种方式正确
  PARTIAL_SCORE: 4,      // 部分得分
  // 两种方式都错误
  WRONG_SCORE: 0,        // 错误得分
  // 快速评分：没印象
  QUICK_NO_IMPRESSION: 0, // 快速评分-无印象
  // 快速评分：有印象但想不起来
  QUICK_SOME_IMPRESSION: 2, // 快速评分-有印象
} as const;
```

**测试方式**：
- 方式1：拼写（最难）
- 方式2：拆分+释义（中等）
- 方式3：识别（最简单）

**得分计算**：
- 两种方式都正确：6分
- 只有一种方式正确：4分
- 两种方式都错误：0分

#### 判断流程图

```
开始复习单词
    ↓
计算稳定性
    ↓
稳定性 ≥ 0.5天？
    ↓ 否
  不满足锁1 → 未掌握
    ↓ 是
获取最近2次得分
    ↓
连续2次 ≥ 5分？
    ↓ 否
  不满足锁2 → 未掌握
    ↓ 是
两条件都满足 → ✓ 已掌握
```

#### 判断示例

| 单词 | 稳定性 | 最近得分 | 锁1 | 锁2 | 结果 |
|------|-------|---------|-----|-----|------|
| apple | 0.6天 | [6, 5] | ✓ | ✓ | 已掌握 |
| book | 0.4天 | [6, 6] | ✗ | ✓ | 未掌握（锁1） |
| ephemeral | 0.8天 | [6, 4] | ✓ | ✗ | 未掌握（锁2） |
| doctor | 0.5天 | [5, 5] | ✓ | ✓ | 已掌握 |

### 3.2 稳定性计算

**计算位置**：`/workspace/projects/client/algorithm/fsrs.ts`

```typescript
function predictNextInterval(word: Word, score: number): number {
  const { stability, difficulty } = word;
  
  // 将得分 (0-6) 映射到回忆质量 (0-5)
  const quality = scoreToQuality(score);
  
  // 基于回忆质量计算新的稳定性
  let newStability: number;
  
  if (quality >= 3) {
    // 回答正确
    const easyBonus = quality === 5 ? 1.2 : (quality === 4 ? 1.1 : 1.0);
    newStability = stability * Math.max(FSRS_PARAMS.EASE_FACTOR, easyBonus * (1 + difficulty * 0.5));
  } else {
    // 回答错误
    newStability = stability * 0.4;
  }
  
  // 确保稳定性在合理范围内
  newStability = Math.max(FSRS_PARAMS.MINIMUM_STABILITY, newStability);
  
  // 计算间隔天数
  const intervalDays = newStability;
  
  return Math.min(FSRS_PARAMS.MAXIMUM_INTERVAL, intervalDays);
}
```

**稳定性计算公式**：

**回答正确（得分≥3）**：
```
新稳定性 = 当前稳定性 × max(1.3, 奖励系数 × (1 + 难度 × 0.5))

奖励系数：
- 得分6分：1.2
- 得分5分：1.2
- 得分4分：1.1
- 得分3分：1.0
```

**回答错误（得分≤2）**：
```
新稳定性 = 当前稳定性 × 0.4
```

**稳定性限制**：
```
新稳定性 = max(0.1, min(新稳定性, 36500))
```

**稳定性更新示例**：

| 当前稳定性 | 得分 | 难度 | 奖励系数 | 新稳定性 |
|-----------|------|------|---------|---------|
| 0.1天 | 6分 | 0.5 | 1.2 × 1.25 = 1.5 | 0.15天 |
| 0.5天 | 5分 | 0.3 | 1.2 × 1.15 = 1.38 | 0.69天 |
| 1天 | 4分 | 0.7 | 1.1 × 1.35 = 1.485 | 1.485天 |
| 3天 | 3分 | 0.5 | 1.0 × 1.25 = 1.25 | 3.75天 |
| 10天 | 0分 | 0.4 | - | 4天 |
| 0.2天 | 6分 | 0.2 | 1.2 × 1.1 = 1.32 | 0.264天 |

### 3.4 可提取性（Retrievability）计算

**计算位置**：`/workspace/projects/client/algorithm/fsrs.ts`

```typescript
export function calculateRetrievability(word: Word): number {
  if (!word.next_review || !word.last_review || word.stability <= 0) {
    return 1.0; // 新单词，可提取性为 1
  }
  
  const lastReview = new Date(word.last_review);
  const now = new Date();
  const elapsedDays = (now.getTime() - lastReview.getTime()) / (1000 * 60 * 60 * 24);
  
  // R = exp(-经过天数 / S)
  const R = Math.exp(-elapsedDays / Math.max(word.stability, 0.1));
  return Math.max(0, Math.min(1, R));
}
```

**公式**：
```
可提取性 R = exp(-经过天数 / 稳定性)
```

**含义**：
- R表示当前时刻能回忆起该单词的概率
- R=1.0：完全能回忆
- R=0.0：完全不能回忆

**可提取性示例**：

| 稳定性 | 经过天数 | 可提取性 | 记忆状态 |
|-------|---------|---------|---------|
| 0.5天 | 0天 | 1.0 | 完全能回忆 |
| 0.5天 | 0.25天 | 0.606 | 部分回忆 |
| 0.5天 | 0.5天 | 0.368 | 记忆下降 |
| 1天 | 0天 | 1.0 | 完全能回忆 |
| 1天 | 0.5天 | 0.606 | 部分回忆 |
| 1天 | 1天 | 0.368 | 记忆下降 |
| 3天 | 0天 | 1.0 | 完全能回忆 |
| 3天 | 1天 | 0.716 | 部分回忆 |
| 3天 | 3天 | 0.368 | 记忆下降 |

**复习触发条件**：
```typescript
export function needsReview(word: Word): boolean {
  const retrievability = calculateRetrievability(word);
  const isMastered = word.is_mastered === 1;
  
  return !isMastered && (retrievability < 0.9 || !word.next_review);
}
```

**说明**：
- 未掌握单词：`is_mastered = 0`
- 可提取性 < 0.9（90%）：需要复习
- 无下次复习时间：立即复习

```typescript
export async function getReviewPlan(daysAhead: number = 7): Promise<ReviewPlanItem[]> {
  const db = getDatabase();
  const now = new Date();
  const endDate = new Date();
  endDate.setDate(now.getDate() + daysAhead);
  
  // 获取所有有复习计划的单词（next_review 不为空）
  const rows = await db.getAllAsync<any>(
    `SELECT w.*, ww.wordbook_id, wb.name as wordbook_name
     FROM words w
     INNER JOIN wordbook_words ww ON w.id = ww.word_id
     INNER JOIN wordbooks wb ON ww.wordbook_id = wb.id
     WHERE w.next_review IS NOT NULL 
     AND w.next_review <= ?
     ORDER BY w.next_review ASC, wb.name ASC`,
    [endDate.toISOString()]
  );
  
  // 按日期分组
  const planMap = new Map<string, Array<{ word: Word; wordbookId: number; wordbookName: string }>>();
  
  rows.forEach(row => {
    const word = mapToWord(row);
    const reviewDate = word.next_review ? new Date(word.next_review) : null;
    
    if (reviewDate) {
      const dateKey = reviewDate.toISOString().split('T')[0];  // YYYY-MM-DD
      
      if (!planMap.has(dateKey)) {
        planMap.set(dateKey, []);
      }
      
      planMap.get(dateKey)!.push({
        word,
        wordbookId: row.wordbook_id,
        wordbookName: row.wordbook_name
      });
    }
  });
  
  // ... 转换为数组并排序 ...
}
```

**复习计划数据结构**：

```typescript
export interface ReviewPlanItem {
  date: string;  // ISO 格式的日期
  dateLabel: string;  // 显示的日期标签，如 "今天"、"明天"、"2024-01-15"
  timeLabel: string;  // 显示的时间标签，如 "10:30"

    wordbookName: string;
    words: Word[];
```

**复习计划生成逻辑**：

1. **筛选单词**：
   - `next_review IS NOT NULL`：有下次复习时间
   - `next_review <= endDate`：在指定时间范围内
   - `ORDER BY next_review ASC`：按时间排序

2. **按日期分组**：
   - 使用日期的 `YYYY-MM-DD` 格式作为分组键
   - 每组包含该日期需要复习的所有单词

3. **按词库分组**：
   - 每个日期组内，按词库ID再次分组
   - 显示每个词库的单词数量

## 总结

### 核心算法流程

1. **单词添加**：
   - 初始稳定性：0天
   - 初始难度：0.5
   - 初始掌握状态：未掌握

2. **首次复习**：
   - 计算得分（0-6分）
   - 计算加权得分（考虑回忆时间）
   - 应用掌握率调整因子（考虑复习时机）
   - 更新难度和稳定性
   - 计算下次复习时间

3. **后续复习**：
   - 每次复习都更新难度、稳定性、下次复习时间
   - 检查是否满足双锁机制（稳定性≥0.5天 + 连续2次≥5分）
   - 满足双锁机制则标记为已掌握

4. **复习计划生成**：
   - 筛选所有未掌握且需要复习的单词（R < 0.9）
   - 按日期分组
   - 按词库分组
   - 显示到复习计划页面

### 关键特性

✅ **动态间隔**：根据用户表现和单词难度调整复习间隔
✅ **精确到小时**：间隔计算精确到小时，而非仅天
✅ **个体差异**：每个用户有独立的复习路径
✅ **时机调整**：提前/延后复习有惩罚机制
✅ **双锁机制**：确保真正掌握才标记为已掌握
✅ **必要难度理论**：在必要难度中实现最佳记忆效果
