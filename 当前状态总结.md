## 当前问题

❌ **在 Expo Go 中无法运行**
- 原因：`react-native-tesseract-ocr` 是原生模块，需要包含在 APK 中
- 错误：`Cannot read property 'recognize' of null`

## 解决方案

### 方案 1：EAS Build（推荐）⭐

**优势**：
- ✅ 无需配置本地环境
- ✅ 使用 Expo 云端构建
- ✅ 10-20 分钟即可获得 APK

**步骤**：
```bash
cd /workspace/projects/client

# 1. 安装 EAS CLI
npm install -g eas-cli

# 2. 登录 Expo
eas login

# 3. 配置项目
eas build:configure

# 4. 构建开发版本
eas build --platform android --profile development

# 5. 下载并安装 APK
```

### 方案 2：本地构建

**优势**：
- ✅ 完全免费
- ✅ 快速迭代

**前置条件**：
- Android Studio
- JDK 17
- Android SDK API 34+

**步骤**：
```bash
cd /workspace/projects/client

# 1. 生成原生项目
npx expo prebuild --platform android --clean

# 2. 构建并安装
npx expo run:android
```

## 注意事项

⚠️ **Expo Go 不支持原生模块**
- 所有自定义原生模块（如 react-native-tesseract-ocr）都无法在 Expo Go 中运行
- 必须使用开发构建（Development Build）或正式构建

⚠️ **首次使用需要联网**
- 首次运行会下载英文语言包（约 10MB）
- 下载完成后可以离线使用

⚠️ **识别速度**
- BASE 模式：2-3 秒（快速）
- BEST 模式：5-8 秒（高精度）
