# 编码记忆法 - 本地打包指南

## 📦 方案 2：本地打包

本指南将帮助你将「编码记忆法」应用打包成 APK（Android）或 IPA（iOS），可以直接安装到手机上使用。

---

## 🖥️ 一、环境要求

### Android 打包要求

| 软件/工具 | 版本要求 | 必需 | 说明 |
|-----------|---------|------|------|
| **Node.js** | >= 18.x | ✅ 必需 | JavaScript 运行环境 |
| **Java JDK** | >= 17 | ✅ 必需 | Android 开发需要 |
| **Android Studio** | 最新版 | ✅ 必需 | Android SDK 和构建工具 |
| **Android SDK** | API Level 34 | ✅ 必需 | Android 目标 SDK |
| **Android SDK Build-Tools** | 34.0.0 | ✅ 必需 | 构建工具 |
| **环境变量** | ANDROID_HOME | ✅ 必需 | Android SDK 路径 |

### iOS 打包要求

| 软件/工具 | 版本要求 | 必需 | 说明 |
|-----------|---------|------|------|
| **Node.js** | >= 18.x | ✅ 必需 | JavaScript 运行环境 |
| **Xcode** | >= 15.x | ✅ 必需 | iOS 开发工具（仅 Mac） |
| **CocoaPods** | >= 1.12 | ✅ 必需 | iOS 依赖管理 |
| **Apple Developer 账号** | 付费或免费 | ⚠️ 建议 | 签名需要（测试版可免） |
| **macOS** | Monterey 或更新 | ✅ 必需 | 仅支持 Mac |

## 🚀 二、准备工作

### 步骤 1：安装 Node.js

#### Windows
1. 访问：https://nodejs.org/
2. 下载 LTS 版本（推荐 18.x 或 20.x）
3. 运行安装程序，一路"下一步"
4. 验证安装：
```bash
node --version
npm --version
```

#### macOS
```bash
# 使用 Homebrew 安装
brew install node

# 验证安装
node --version
npm --version
```

#### Linux (Ubuntu/Debian)
```bash
# 使用 NodeSource 安装
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# 验证安装
node --version
npm --version
```

---

### 步骤 2：安装 Java JDK

#### Windows
1. 访问：https://adoptium.net/
2. 下载 Temurin JDK 17（或 21）
3. 运行安装程序
4. 配置环境变量：
   - 右键"此电脑" → 属性 → 高级系统设置 → 环境变量
   - 新建系统变量：`JAVA_HOME`，值为 JDK 安装路径（如：`C:\Program Files\Eclipse Adoptium\jdk-17.0.9.101-hotspot`）
   - 编辑 `Path` 变量，添加 `%JAVA_HOME%\bin`

#### macOS
```bash
# 使用 Homebrew 安装
brew install openjdk@17

# 配置环境变量（添加到 ~/.zshrc 或 ~/.bash_profile）
echo 'export JAVA_HOME=$(/usr/libexec/java_home -v 17)' >> ~/.zshrc
source ~/.zshrc

# 验证安装
java -version
```

#### Linux (Ubuntu/Debian)
```bash
sudo apt-get update
sudo apt-get install -y openjdk-17-jdk

# 验证安装
java -version
```

---

### 步骤 3：安装 Android Studio（仅 Android）

1. 访问：https://developer.android.com/studio
2. 下载并安装 Android Studio
3. 启动 Android Studio，完成初始设置向导
4. 在向导中安装以下组件：
   - Android SDK Platform 34
   - Android SDK Build-Tools 34.0.0
   - Android Emulator（可选，用于测试）
   - Android SDK Platform-Tools

5. 配置环境变量：

   **Windows:**
   - `ANDROID_HOME` = `C:\Users\你的用户名\AppData\Local\Android\Sdk`
   - `Path` 添加：
     - `%ANDROID_HOME%\platform-tools`
     - `%ANDROID_HOME%\emulator`
     - `%ANDROID_HOME%\tools`
     - `%ANDROID_HOME%\tools\bin`

   **macOS / Linux:**
   ```bash
   # 添加到 ~/.zshrc 或 ~/.bash_profile
   export ANDROID_HOME=$HOME/Android/Sdk
   export PATH=$PATH:$ANDROID_HOME/platform-tools
   export PATH=$PATH:$ANDROID_HOME/emulator
   export PATH=$PATH:$ANDROID_HOME/tools
   export PATH=$PATH:$ANDROID_HOME/tools/bin
   source ~/.zshrc

   # 验证
   adb version
   ```

---

### 步骤 4：安装 CocoaPods（仅 iOS）

```bash
# 使用 RubyGems 安装（macOS 自带 Ruby）
sudo gem install cocoapods

# 验证安装
```

## 🔨 五、打包步骤

### Android 打包（生成 APK）

#### 方法 1：使用 Development Build（推荐）

```bash
# 1. 生成原生代码（Prebuild）
npx expo prebuild --clean

# 2. 进入 Android 项目目录
cd android

# 3. 使用 Gradle 构建 Debug APK
./gradlew assembleDebug

# 或者构建 Release APK（需要签名）
./gradlew assembleRelease

# 4. APK 文件位置
# Debug: android/app/build/outputs/apk/debug/app-debug.apk
# Release: android/app/build/outputs/apk/release/app-release.apk
```

#### 方法 2：使用 EAS Build（云端构建，简单）

```bash
# 1. 初始化 EAS 配置
eas build:configure

# 2. 构建 APK（自动签名）
eas build --platform android --profile preview

# 3. 下载 APK
# 构建完成后，EAS 会提供下载链接
```

#### 方法 3：使用 Expo Application Services（最简单）

```bash
# 1. 构建 Standalone App
eas build --platform android --local

# 2. APK 文件会自动下载到本地
```

---

### iOS 打包（生成 IPA）

#### 前提条件
- Mac 电脑
- Xcode 15+
- Apple Developer 账号（免费或付费）

#### 方法 1：使用 Development Build（推荐）

```bash
# 1. 生成原生代码（Prebuild）
npx expo prebuild --clean

# 2. 进入 iOS 项目目录
cd ios

# 3. 安装 CocoaPods 依赖
pod install

# 4. 使用 Xcode 打开项目
open wordReview.xcworkspace

# 5. 在 Xcode 中：
#    - 选择你的 Team（Apple Developer 账号）
#    - 选择目标设备（真机）
#    - 点击 Product → Archive

# 6. 导出 IPA
#    - Archive 完成后，点击 Distribute App
#    - 选择 Ad Hoc 或 Enterprise
#    - 导出 IPA 文件
```

#### 方法 2：使用 EAS Build（云端构建）

```bash
# 1. 配置 Apple 凭证
eas credentials

# 2. 构建 IPA
eas build --platform ios --profile preview

# 3. 下载 IPA
```

## ⚠️ 七、常见问题解决

### 问题 1：`ANDROID_HOME` 环境变量未设置

**错误信息**：
```
ANDROID_HOME is not set
```

**解决方法**：
```bash
# Windows
setx ANDROID_HOME "C:\Users\你的用户名\AppData\Local\Android\Sdk"

# macOS / Linux
export ANDROID_HOME=$HOME/Android/Sdk
echo 'export ANDROID_HOME=$HOME/Android/Sdk' >> ~/.zshrc
```

---

### 问题 2：Java 版本不匹配

**错误信息**：
```
Unsupported class file major version 61
```

**解决方法**：
```bash
# 安装 Java 17
java -version  # 检查版本

# 如果版本不对，卸载后重新安装
```

---

### 问题 3：Gradle 构建失败

**错误信息**：
```
FAILURE: Build failed with an exception.
```

**解决方法**：
```bash
# 清理 Gradle 缓存
cd android
./gradlew clean

# 删除 .gradle 文件夹
rm -rf ~/.gradle

# 重新构建
./gradlew assembleDebug
```

---

### 问题 4：iOS CocoaPods 安装失败

**错误信息**：
```
[!] Unable to find a specification for `expo`
```

**解决方法**：
```bash
# 更新 CocoaPods 仓库
pod repo update

# 清理 CocoaPods 缓存
pod cache clean --all

# 重新安装
cd ios
pod install
```

---

### 问题 5：APK 安装失败

**错误信息**：
```
INSTALL_FAILED_UPDATE_INCOMPATIBLE
```

**解决方法**：
```bash
# 卸载旧版本
adb uninstall com.wordreview.app

# 重新安装
adb install android/app/build/outputs/apk/debug/app-debug.apk
```
