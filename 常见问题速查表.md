## 📥 下载项目相关

### 问题：找不到项目文件夹在哪里

**解决方法：**

**如果你刚下载：**
```
→ 打开 "下载" 文件夹
→ 应该能看到 projects.zip 或类似的压缩包
→ 右键点击 → 解压
→ 解压后得到 projects 文件夹
```

**如果你已经解压：**
```
→ 按 Win + E 打开文件管理器
→ 在左侧栏找 "下载"
→ 找到 projects 文件夹
```

**实在找不到：**
```
→ 按 Win + S
→ 输入 "projects"
→ 看看搜索结果
```

## 💻 安装依赖相关

### 问题：提示 "pnpm 不是内部或外部命令"

**解决方法：**
```
1. 在命令窗口输入：npm install -g pnpm
2. 按回车
3. 等待安装完成
4. 然后再执行 pnpm install
```

### 问题：提示 "node 不是内部或外部命令"

**解决方法：**
```
1. 打开浏览器，访问：https://nodejs.org/
2. 下载 LTS 版本（左边那个，推荐下载）
3. 双击下载的安装包
4. 一路点击"下一步"或"Next"
5. 安装完成后重启电脑
6. 重新打开命令窗口执行
```

### 问题：安装过程卡住不动

**解决方法：**
```
1. 检查网络是否正常
2. 关闭 VPN 试试
3. 如果还是卡住：
   - 按 Ctrl + C 停止
   - 重新执行 pnpm install
4. 多试几次
```

## 🏗️ 生成 Android 项目相关

### 问题：提示 "expo 不是内部或外部命令"

**解决方法：**
```
1. 确保在 client 文件夹里
   - 输入：cd client
   - 按回车

2. 安装 expo
   - 输入：pnpm add expo
   - 按回车

3. 重新执行
   - 输入：npx expo prebuild --platform android --clean
   - 按回车
```

### 问题：提示 "Java 版本不匹配"

**解决方法：**
```
1. 检查 Java 版本
   - 输入：java -version
   - 按回车

2. 应该显示 17.x.x
   - 如果不是，需要安装 JDK 17

3. 下载 JDK 17
   - 访问：https://adoptium.net/
   - 选择 JDK 17
   - 下载并安装

4. 安装后重启电脑
```

### 问题：生成失败，提示权限错误

**解决方法：**
```
→ 以管理员身份运行命令提示符
→ 右键点击"命令提示符"
→ 选择"以管理员身份运行"
→ 重新执行命令
```

## 🎨 Android Studio 相关

### 问题：找不到 Android Studio

**解决方法：**
```
1. 按 Win 键
2. 输入 "Android Studio"
3. 如果看到了，点击打开
4. 如果看不到：
   - 重新安装 Android Studio
   - 访问：https://developer.android.com/studio
```

### 问题：Gradle 同步一直失败

**解决方法：**
```
1. 检查网络
   - 确保能访问外网
   - 关闭 VPN

2. 清理缓存
   - File → Invalidate Caches
   - 点击 "Invalidate and Restart"

3. 重新打开项目

4. 如果还是失败
   - 查看 Build 窗口的错误信息
   - 截图发给懂技术的人
```

### 问题：Gradle 同步太慢，超过 30 分钟

**解决方法：**
```
1. 这是正常的，第一次就这样
2. 可以去做别的事情
3. 不要关闭 Android Studio
4. 等待同步完成
5. 如果超过 1 小时：
   - 关闭 Android Studio
   - 重新打开项目
   - 再次等待同步
```

### 问题：提示 "SDK 未找到"

**解决方法：**
```
1. Tools → SDK Manager
2. 勾选 "Android 14.0 (API 34)"
3. 点击 "Apply" 或 "OK"
4. 等待下载完成
5. 重新同步项目
```

## 📱 手机连接相关

### 问题：找不到开发者选项

**解决方法：**
```
→ 先开启开发者模式：
   设置 → 关于手机 → 连续点击"版本号" 7 次

→ 然后就能看到开发者选项了：
   设置 → 更多设置 → 开发者选项
```

### 问题：手机上没有弹出授权窗口

**解决方法：**
```
1. 检查 USB 调试是否开启
   - 设置 → 开发者选项
   - 确保 "USB 调试" 已打开

2. 重新插拔 USB 线

3. 关闭再重新开启 USB 调试

4. 换一个 USB 接口试试
```

### 问题：Android Studio 看不到我的手机

**解决方法：**
```
1. 在 Android Studio 打开 Terminal（底部标签）
2. 输入：adb devices
3. 按回车

4. 如果能看到设备：
   - 重启 Android Studio
   - 应该就能看到了

5. 如果看不到：
   - 重新插拔 USB 线
   - 检查 USB 调试是否开启
   - 换一根 USB 线
```

### 问题：手机一直显示"正在通过 USB 调试"

**解决方法：**
```
→ 这是正常的，说明已经连接成功
→ 只要在 Android Studio 能看到设备就可以
→ 可以继续下一步操作
```

## ▶️ 构建运行相关

### 问题：点击运行按钮没反应

**解决方法：**
```
1. 检查是否选择了设备
   - 看看 Android Studio 顶部工具栏
   - 应该显示你的手机型号
   - 如果是 "No devices"，说明手机没连接

2. 检查手机是否解锁
   - 确保手机屏幕是亮着的
   - 没有被锁定

3. 重新点击运行按钮
```

### 问题：构建失败，提示各种错误

**解决方法：**
```
1. 查看 Build 窗口的错误信息
   - 点击 Android Studio 底部的 "Build" 标签
   - 滚动到最下面看红色错误信息

2. 常见错误类型：
   - 语法错误：检查代码
   - 依赖错误：检查网络
   - 配置错误：检查配置文件

3. 尝试清理后重新构建：
   - Build → Clean Project
   - Build → Rebuild Project

4. 如果还是失败：
   - 截图错误信息
   - 找懂技术的人帮忙
```

### 问题：构建成功但应用没安装到手机

**解决方法：**
```
1. 在 Terminal 输入：cd android
2. 输入：./gradlew installDebug
3. 按回车
4. 等待安装完成
5. 在手机上手动打开应用
```

## 🧪 测试识别相关

### 问题：应用打开后闪退

**解决方法：**
```
1. 查看日志
   - View → Tool Windows → Logcat
   - 看看有什么错误

2. 常见原因：
   - 权限未授予（相机权限）
   - 配置错误
   - 代码错误

3. 解决方法：
   - 重新构建一次
   - 检查 Build 窗口的警告
```

### 问题：识别失败，提示 "ML Kit 模块未加载"

**解决方法：**
```
→ 这说明你用的不是原生构建的应用

→ 确认你：
   1. 是通过 Android Studio 运行的应用
   2. 不是通过 Expo Go 打开的
   3. 应用图标旁边没有 "Go" 字样

→ 重新构建一次：
   cd android
   ./gradlew installDebug
```

### 问题：能识别但准确率很低

**解决方法：**
```
→ 这是正常现象，原因可能是：
   1. 拍摄不清晰
   2. 光线不足
   3. 角度不对
   4. 单词太小

→ 改进方法：
   1. 保证拍摄清晰
   2. 充足光线
   3. 正对拍摄
   4. 距离适中
```

## 🔄 重新构建相关

### 问题：修改代码后如何重新安装

**解决方法：**
```
1. 在 Android Studio 打开 Terminal
2. 输入：cd android
3. 输入：./gradlew installDebug
4. 按回车
5. 等待完成（1-2 分钟）
6. 在手机上重新打开应用
```

### 问题：如何完全重新构建

**解决方法：**
```
1. 在 Android Studio
   - Build → Clean Project

2. 在 Terminal
   - cd android
   - ./gradlew clean

3. 重新构建
   - Build → Rebuild Project

或者：
   - 点击运行按钮重新构建
```

## 📱 获取 APK 文件

### 问题：APK 文件在哪里

**解决方法：**
```
Debug APK 位置：
client/android/app/build/outputs/apk/debug/app-debug.apk

Release APK 位置：
client/android/app/build/outputs/apk/release/app-release.apk
```

### 问题：如何分享 APK 给别人

**解决方法：**
```
1. 找到 APK 文件（上面有路径）
2. 复制到桌面
3. 通过微信/QQ 发给朋友
4. 朋友下载后安装即可
```


## 💡 重要提示

1. **不要着急**：第一次构建慢是正常的
2. **保持网络**：构建过程需要联网
3. **不要关闭**：Android Studio 构建期间不要关闭
4. **耐心等待**：Gradle 同步可能需要 10-30 分钟
5. **截图保存**：遇到问题先截图错误信息
