# 🎯 电脑小白专用：Android 本地构建超详细指南

## 📱 前言

这份指南专门写给**完全不懂电脑操作**的用户，每一步都有详细说明，跟着做就行了！

---

## 🛠️ 准备工作（必须完成）

### ✅ 确认你有以下东西：

1. ✅ 一台 **Windows 或 macOS** 电脑
2. ✅ **Android Studio** 已安装（你说已安装，那就好了）
3. ✅ **JDK 17** 已安装（你说已安装，那就好了）
4. ✅ 一根 **USB 数据线**（连接手机和电脑用）
5. ✅ 一台 **Android 手机**（用于测试）

## 📥 第一步：下载项目代码（3 分钟）

### 🌐 方法 A：如果你能看到项目文件夹（推荐）

1. 在你的电脑上，找到项目代码存放的位置（比如：`D:\workspace\projects`）
2. 确认里面有一个 `client` 文件夹
3. ✅ 如果能看到，**跳到第二步**

### 🌐 方法 B：如果你需要下载项目代码

**情况 1：项目在云盘（百度网盘、阿里云盘等）**

```
1. 打开云盘网页或客户端
2. 找到项目压缩包（比如 projects.zip）
3. 下载到你的电脑桌面
4. 右键点击压缩包 → "解压到当前文件夹"
5. 解压后得到 projects 文件夹
```

**情况 2：项目在 GitHub 或其他代码网站**

```
1. 打开网页
2. 找到绿色的 "Code" 或 "下载" 按钮
3. 点击 "Download ZIP"
4. 等待下载完成
5. 打开 "下载" 文件夹
6. 右键点击下载的 ZIP 文件
7. 选择 "解压到当前文件夹"
```

**情况 3：有人直接给你发文件夹**

```
1. 打开微信/QQ 的文件接收窗口
2. 找到项目文件夹
3. 右键点击 → "复制"
4. 打开电脑的 D 盘
5. 右键点击空白处 → "粘贴"
```

### ✅ 完成标志

在你的电脑上，你能找到这样的文件夹结构：

```
📁 projects (主文件夹)
  ├─ 📁 client (这个文件夹很重要！)
  ├─ 📁 server (可能没有)
  ├─ 📄 app.config.ts
  ├─ 📄 package.json
  └─ 其他文件...
```

## 💻 第二步：安装依赖（5 分钟）

### 🎯 目标

让电脑下载项目需要的所有程序包。

### 📝 操作步骤

#### Windows 用户：

1. **打开命令提示符**
   - 按 `Win + R` 键（Win 键就是有微软标志的那个键）
   - 输入 `cmd`
   - 按 `回车` 键
   - 会出来一个黑色的窗口

2. **进入项目文件夹**
   - 在黑色窗口里，输入：
     ```
     cd D:\workspace\projects\client
     ```
   - 注意：如果你的项目不在这个位置，改成你的实际路径
   - 按 `回车` 键

3. **安装依赖**
   - 输入：
     ```
     pnpm install
     ```
   - 按 `回车` 键
   - **等待完成**（可能需要 2-5 分钟）
   - 看到 `Done in xx seconds` 就完成了

#### macOS 用户：

1. **打开终端**
   - 按 `Command + 空格` 键
   - 输入 `终端`
   - 按 `回车` 键

2. **进入项目文件夹**
   - 输入：
     ```
     cd ~/Desktop/projects/client
     ```
   - 注意：如果项目在桌面，就用这个命令
   - 按 `回车` 键

3. **安装依赖**
   - 输入：
     ```
     pnpm install
     ```
   - 按 `回车` 键
   - **等待完成**（可能需要 2-5 分钟）

### ❌ 如果出错

**错误 1：提示 "pnpm 不是内部或外部命令"**

解决方法：
1. 先安装 pnpm：
   ```
   npm install -g pnpm
   ```
2. 然后再执行 `pnpm install`

**错误 2：提示 "node 不是内部或外部命令"**

解决方法：
1. 去这个网站下载 Node.js：https://nodejs.org/
2. 下载 LTS 版本（长期支持版）
3. 双击安装，一路点"下一步"
4. 安装完成后重启电脑
5. 重新执行上面的命令

### ✅ 完成标志

看到这样的提示：
```
Done in 3.2s
```

就成功了！

## 🏗️ 第三步：生成 Android 项目（5 分钟）

### 📝 操作步骤

继续在上面的命令窗口里操作：

1. **生成 Android 项目**
   - 输入：
     ```
     npx expo prebuild --platform android --clean
     ```
   - 按 `回车` 键
   - **等待完成**（可能需要 2-5 分钟）

2. **看到成功提示**
   - 会提示类似：
     ```
     ✅ Generated native Android project successfully!
     ```
   - 这时候你的 `client` 文件夹里多了一个 `android` 文件夹

### ❌ 如果出错

**错误 1：提示 "expo 不是内部或外部命令"**

这是因为没有安装 expo，执行：
```
cd client
pnpm add expo
```
然后再重新执行 `npx expo prebuild`

**错误 2：提示 "Java 版本不匹配"**

确保安装了 JDK 17：
- 检查版本：`java -version`
- 应该显示 `17.x.x`

### ✅ 完成标志

1. 你的 `client` 文件夹里多了一个 `android` 文件夹
2. 打开 `android` 文件夹，能看到很多文件和文件夹

---

## 🎨 第四步：用 Android Studio 打开项目（5 分钟）

### 📝 操作步骤

1. **打开 Android Studio**
   - 在电脑桌面找到 Android Studio 图标
   - 双击打开
   - 如果没有图标，按 `Win` 键，输入 "Android Studio"，打开

2. **打开项目**
   - 点击 `File` 菜单（在左上角）
   - 选择 `Open...`
   - 在弹出的窗口里：
     - 找到 `client/android` 文件夹
     - 点击选中 `android` 文件夹（不是里面的文件）
     - 点击右下角的 `OK` 按钮

3. **等待 Gradle 同步**
   - Android Studio 会自动下载依赖
   - 底部会显示进度条
   - **第一次可能需要 10-30 分钟**
   - ⚠️ 期间不要关闭电脑和网络
   - 等到底部显示 `Build completed successfully` 就完成了

### 💡 提示

- 同步期间可以去喝杯茶 ☕
- 如果长时间不动（超过 1 小时），可以：
  1. 关闭 Android Studio
  2. 重新打开项目
  3. 再次等待同步

### ❌ 如果出错

**错误 1：提示 "Gradle sync failed"**

解决方法：
1. 点击底部的 `Build` 标签
2. 查看具体错误信息
3. 通常是网络问题，确保能访问外网
4. 关闭 VPN 试试

**错误 2：提示 "SDK 未找到"**

解决方法：
1. 点击 `Tools > SDK Manager`
2. 勾选 `Android 14.0 (API 34)`
3. 点击 `Apply` 或 `OK`
4. 等待下载完成

### ✅ 完成标志

1. Android Studio 底部显示绿色对勾
2. 提示 `BUILD SUCCESSFUL`
3. 顶部工具栏能看到你的设备（如果已连接手机）

## 📱 第五步：连接 Android 手机（5 分钟）

### 📝 操作步骤

#### 1. 开启手机的开发者模式

**小米手机：**
```
1. 打开 "设置"
2. 找到 "我的设备"
3. 连续点击 "MIUI 版本" 7 次
4. 看到 "已处于开发者模式" 提示
5. 返回设置，找到 "更多设置 > 开发者选项"
```

**华为/荣耀手机：**
```
1. 打开 "设置"
2. 找到 "关于手机"
3. 连续点击 "版本号" 7 次
4. 看到 "已处于开发者模式" 提示
5. 返回设置，找到 "系统和更新 > 开发者选项"
```

**OPPO/Vivo 手机：**
```
1. 打开 "设置"
2. 找到 "关于手机"
3. 连续点击 "版本号" 7 次
4. 看到 "已处于开发者模式" 提示
5. 返回设置，找到 "其他设置 > 开发者选项"
```

**三星手机：**
```
1. 打开 "设置"
2. 找到 "关于手机"
3. 点击 "软件信息"
4. 连续点击 "版本号" 7 次
5. 返回设置，找到 "开发者选项"
```

#### 2. 开启 USB 调试

```
1. 打开 "开发者选项"
2. 找到 "USB 调试"
3. 打开开关（变成绿色）
```

#### 3. 连接手机和电脑

```
1. 用 USB 数据线连接手机和电脑
2. 手机会弹出授权窗口
3. 勾选 "始终允许使用这台电脑进行调试"
4. 点击 "允许" 或 "OK"
```

#### 4. 在 Android Studio 验证连接

```
1. 看看 Android Studio 顶部工具栏
2. 应该能看到你的手机型号
3. 如果看不到，点击旁边的设备下拉箭头
4. 选择你的手机
```

### ❌ 如果连接不上

**问题 1：电脑不识别手机**

解决方法：
1. 更换一根 USB 数据线
2. 尝试不同的 USB 接口
3. 重新插拔 USB 线

**问题 2：手机上没有授权提示**

解决方法：
1. 检查 USB 调试是否开启
2. 关闭再重新开启 USB 调试
3. 重新插拔 USB 线

**问题 3：Android Studio 看不到设备**

解决方法：
1. 在 Android Studio 打开命令行（`Terminal` 标签）
2. 输入 `adb devices`
3. 如果看到你的设备，说明连接成功
4. 如果没有，尝试：
   ```
   adb kill-server
   adb start-server
   ```

### ✅ 完成标志

1. Android Studio 顶部工具栏显示你的手机型号
2. 手机显示 "正在通过 USB 调试" 或类似提示

## ▶️ 第六步：构建并安装应用（5-10 分钟）

### 📝 操作步骤

1. **选择运行配置**
   - 在 Android Studio 顶部工具栏
   - 确保选择了你的手机
   - 点击绿色播放按钮 ▶️（Run 按钮）

2. **等待构建和安装**
   - Android Studio 会自动编译代码
   - 打包成 APK
   - 安装到你的手机
   - **可能需要 3-10 分钟**

3. **应用自动打开**
   - 安装完成后，应用会在手机上自动打开
   - 🎉 恭喜你，成功了！

### 💡 第一次构建慢的原因

- 需要编译所有原生代码
- 需要打包资源文件
- 需要生成签名文件

### ✅ 完成标志

1. Android Studio 底部显示 `BUILD SUCCESSFUL`
2. 手机上出现应用图标
3. 应用自动打开

## 🧪 第七步：测试本地 OCR（3 分钟）

### 📝 操作步骤

1. **打开应用**
   - 在手机上点击应用图标
   - 等待应用启动

2. **进入相机扫描**
   - 找到 "拍照识别" 或 "相机扫描" 按钮
   - 点击进入

3. **拍照测试**
   - 找一个英文单词（比如书上、卡片上的）
   - 对准单词，放在扫描框内
   - 点击拍照按钮

4. **查看结果**
   - 等待 1-2 秒
   - 应该能看到识别结果
   - ✅ 成功识别！

### 🎉 完美！

如果成功识别，说明：
- ✅ 本地 ML Kit OCR 工作正常
- ✅ 完全离线，不需要网络
- ✅ 识别率 95%+

### ❌ 如果识别失败

**错误：提示 "ML Kit 模块未加载"**

这说明用的是 Expo Go，不是原生构建的应用：
1. 确认你是通过 Android Studio 运行的应用
2. 不是通过 Expo Go 打开的
3. 重新构建一次试试

## 🔄 以后修改代码怎么办？

### 快速重新构建（1-2 分钟）

1. 修改代码
2. 在 Android Studio 打开命令行（`Terminal` 标签）
3. 输入：
   ```
   cd android
   ./gradlew installDebug
   ```
4. 等待完成
5. 在手机上重新打开应用

## 📚 常见问题快速解决

| 问题 | 解决方法 |
|------|---------|
| 找不到项目文件夹 | 检查下载位置，通常在桌面或下载文件夹 |
| pnpm 命令不可用 | 执行 `npm install -g pnpm` |
| Gradle 同步太慢 | 等待，不要关闭，可以去忙别的事 |
| 手机连不上 | 更换 USB 线，检查 USB 调试是否开启 |
| 构建失败 | 查看底部 Build 窗口的错误信息 |
