## 复习分数提交失败问题诊断与修复

### 问题分析

根据代码检查，发现了一个关键问题：

**问题：submitQuickScore 函数中 addReviewLog 调用缺少完整数据**

在 `submitQuickScore` 函数中，调用 `addReviewLog` 时缺少了以下字段：
- `stability_before`
- `stability_after` 
- `difficulty_before`
- `difficulty_after`

虽然这些字段在数据库中是可选的（REAL类型，可以为NULL），但如果数据库迁移没有正确执行，或者某些情况下数据验证更严格，可能会导致插入失败。

### 已修复

**修复位置**: `client/screens/review-detail/index.tsx`

**修复内容**: 为 `submitQuickScore` 函数中的 `addReviewLog` 调用添加完整数据

```typescript
await addReviewLog({
  word_id: word.id,
  score: finalScore,
  response_time: responseTime,
  stability_before: word.stability,        // 新增
  stability_after: newStability,           // 新增
  difficulty_before: word.difficulty,      // 新增
  difficulty_after: newDifficulty,         // 新增
  reviewed_at: new Date().toISOString()
});
```

### 其他可能的原因

1. **数据库迁移问题**
   - 如果 review_logs 表没有正确创建，会导致插入失败
   - 解决方案：检查数据库版本是否正确升级到 7

2. **并发问题**
   - 如果用户快速点击按钮，可能导致多个提交同时进行
   - 解决方案：添加防重复提交的逻辑

3. **SQLite 锁问题**
   - 在某些情况下，SQLite 可能会有锁冲突
   - 解决方案：使用事务处理多个操作

### 建议的改进

1. **添加防止重复提交的逻辑**
   ```typescript
   const [isSubmitting, setIsSubmitting] = useState(false);
   
   const submitQuickScore = async (word: Word, score: number) => {
     if (isSubmitting) return; // 防止重复提交
     setIsSubmitting(true);
     try {
       // ... 提交逻辑
     } finally {
       setIsSubmitting(false);
     }
   };
   ```

2. **使用事务处理**
   ```typescript
   await db.withTransactionAsync(async () => {
     await addReviewLog(...);
     await updateWord(...);
   });
   ```

3. **更详细的错误日志**
   ```typescript
   catch (error) {
     console.error('[Review] 提交分数失败:', {
       wordId: word.id,
       word: word.word,
       score: finalScore,
       error: error.message,
       stack: error.stack
     });
     Alert.alert('提交失败', `错误：${error.message}`);
   }
   ```
