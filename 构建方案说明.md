## 🚀 如何使用（3步）

### 第一步：安装 Docker

1. 下载：https://www.docker.com/products/docker-desktop/
2. 安装后启动 Docker Desktop
3. 确保 Docker 正在运行

### 第二步：运行构建脚本

在 CMD 或 PowerShell 中执行：

```bash
cd D:\workspace\projects
build-apk-docker.bat
```

### 第三步：获取 APK

构建完成后：
- APK 自动打开所在目录
- APK 位置：`D:\workspace\projects\client\android\app\build\outputs\apk\debug\app-debug.apk`

## 🔧 构建脚本说明

### `build-apk-docker.bat`（Windows）

自动执行以下步骤：
1. 检查 Docker 是否运行
2. 构建 Docker 镜像（首次）
3. 清理旧的构建
4. 安装依赖
5. 生成 Android 项目
6. 构建 APK
7. 打开 APK 所在目录

### `Dockerfile.android`

包含完整的 Android 构建环境：
- Node.js 18
- Java 17
- Android SDK 34
- Android Build Tools 34.0.0
- Android NDK 25.1.8937393
- Expo CLI
- EAS CLI

---

## ⚠️ 注意事项

### 1. Docker 空间要求

- Docker 镜像：约 3-5 GB
- 构建缓存：约 5-10 GB
- 总计：约 10-15 GB

### 2. 首次构建较慢

首次构建需要：
- 下载 Docker 镜像
- 下载 Android SDK
- 下载 NDK

请耐心等待，后续构建会快很多。

### 3. 网络要求

构建过程中需要下载：
- Docker 镜像（约 3-5 GB）
- Android SDK（约 500 MB）
- NPM 依赖（约 200 MB）

建议使用稳定网络。

## 📞 遇到问题？

### 常见问题

**Q1: Docker 未运行**
- 启动 Docker Desktop
- 等待托盘图标变为绿色

**Q2: 构建失败**
- 检查网络连接
- 清理 Docker 缓存：`docker system prune -a`
- 重新运行构建脚本

**Q3: APK 安装失败**
- 先卸载旧版本
- 重新安装
