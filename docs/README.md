## 文档分类

### 法律文档 (`legal/`)

#### 1. 隐私政策 (`privacy-policy.md`)

**用途**：说明应用如何收集、使用和保护用户数据

**核心内容**：
- 数据收集声明（本应用不收集任何个人信息）
- 数据使用说明
- 数据安全措施
- 用户权利
- 第三方服务声明

**何时更新**：
- 新增数据收集功能时
- 修改数据处理方式时
- 每年至少审查一次

#### 2. 用户协议 (`terms-of-service.md`)

**用途**：规范用户使用应用的权利和义务

**核心内容**：
- 用户权利与义务
- 知识产权声明
- 免责声明
- 禁止行为
- 争议解决

**何时更新**：
- 新增功能或服务时
- 修改商业模式时
- 法律法规变化时

#### 3. 应用权限说明 (`permissions.md`)

**用途**：详细说明应用请求的各项系统权限及其用途

**核心内容**：
- Android 权限列表
- iOS 权限列表
- 权限请求时机
- 权限使用原则

**何时更新**：
- 新增权限请求时
- 修改权限用途时

## 快速开始

### 1. 准备发布应用

如果您打算发布应用，请按以下步骤操作：

1. **阅读法律文档**
   - 熟悉隐私政策和用户协议的内容
   - 理解应用权限的使用场景

2. **自定义文档**
   - 替换所有占位符（如联系邮箱、开发者名称）
   - 根据应用实际情况调整内容
   - 咨询法律专业人士（推荐）

3. **发布文档**
   - 选择发布方式（在线文档或应用内展示）
   - 更新应用中的文档链接
   - 确保链接可访问

4. **应用商店准备**
   - 准备隐私政策 URL（必需）
   - 填写权限使用说明
   - 提交数据安全声明

### 2. 维护文档

定期审查和更新文档：

- **每季度**：检查文档内容是否准确
- **每年**：全面审查所有法律文档
- **功能更新时**：同步更新相关文档

## 法律文档使用方式

### 在线文档（推荐）

将文档托管到您的网站，通过链接访问。

**优点**：
- 易于更新和管理
- 减轻应用体积
- 便于搜索引擎索引

**步骤**：
1. 将 Markdown 文档转换为 HTML
2. 部署到您的网站或文档平台
3. 更新应用中的链接

### 应用内展示

将文档打包到应用中，在应用内展示。

**优点**：
- 无需网络即可查看
- 更好的用户体验

**步骤**：
1. 将文档复制到应用资源目录
2. 使用 Markdown 渲染库展示
3. 在应用内集成文档查看器

## 常见问题

### Q: 为什么需要这些文档？

**A**:
- 应用商店审核要求
- 用户信任和透明度
- 法律合规（GDPR、CCPA 等）
- 避免法律纠纷

### Q: 我可以直接使用这些模板吗？

**A**: 可以，但必须：
- 根据实际情况修改内容
- 替换所有占位符
- 咨询法律专业人士（强烈推荐）

### Q: 应用完全离线，还需要隐私政策吗？

**A**: 需要。即使不收集数据，仍需：
- 说明数据存储位置
- 告知用户数据使用方式
- 符合应用商店要求

### Q: 如何处理用户反馈？

**A**:
- 在"关于"页面提供反馈入口
- 通过邮件或应用内反馈收集意见
- 及时回复和处理用户问题
