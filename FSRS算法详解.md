## 算法简介

### 什么是 FSRS？

**FSRS（Free Spaced Repetition Scheduler）** 是一种新一代的间隔重复算法，由 **OpenSpacedRepetition** 社区开发，旨在解决传统间隔重复算法的局限性。

**核心目标：**
- 精准预测用户记忆状态
- 个性化调整复习间隔
- 最大化学习效率
- 最小化遗忘率

### 为什么叫"Free"？

**"Free" 代表：**
- **自由**：不受固定间隔限制
- **灵活**：根据个人记忆特点动态调整
- **开放**：算法完全开源，社区共建

## 背景与发展

### 传统算法的局限性

#### 1. 艾宾浩斯曲线（Ebbinghaus Forgetting Curve）

**简介：**
- 1885年，德国心理学家艾宾浩斯提出
- 描述人类记忆随时间衰减的规律
- 复习间隔固定：1天、3天、7天、15天、30天...

**缺点：**
- ❌ 所有用户使用相同的间隔
- ❌ 不考虑个体差异
- ❌ 不考虑单词难度差异
- ❌ 不考虑记忆表现差异

#### 2. SuperMemo 算法

**简介：**
- 1987年，Piotr Wozniak 开发
- 第一个计算机化的间隔重复系统
- 使用复杂公式计算下次复习时间

**缺点：**
- ❌ 参数固定，难以优化
- ❌ 难以适应不同学习风格
- ❌ 数学模型过于复杂

#### 3. Anki 算法

**简介：**
- 基于改进的 SuperMemo 算法
- 每张卡片独立计算复习时间

**缺点：**
- ❌ 仍然使用固定间隔
- ❌ 不考虑学习上下文
- ❌ 难以准确预测记忆状态

## 核心原理

### 记忆的三大要素

FSRS 基于记忆科学的三大核心要素：

#### 1. **稳定性（Stability）**

**定义：**
- 记忆保持不变的持续时间
- 稳定性越高，记忆越持久

**影响因素：**
- 复习次数
- 复习间隔
- 记忆强度

**示例：**
- 首次学习：稳定性 = 1天
- 第1次复习：稳定性 = 3天
- 第2次复习：稳定性 = 10天
- 第3次复习：稳定性 = 30天

---

#### 2. **难度（Difficulty）**

**定义：**
- 单词或卡片的内在难度
- 难度越高，需要更多复习

**影响因素：**
- 单词长度
- 单词复杂度
- 用户熟悉程度

**评分标准：**
- 1-10分，分数越高越难

---

#### 3. **检索率（Retrievability）**

**定义：**
- 当前时刻能够回忆起单词的概率
- 检索率随时间衰减

**计算公式：**
```
检索率 = e^(-t / 稳定性)
```

其中：
- `t`：从上次复习到现在的时长
- `稳定性`：记忆的稳定性

**示例：**
- 刚复习完：检索率 = 100%
- 1天后：检索率 = 36.8%（稳定性=1天）
- 3天后：检索率 = 22.3%（稳定性=3天）

---

### FSRS 的工作流程

#### 步骤 1：学习新单词

**用户操作：**
1. 添加新单词到学习列表
2. 查看单词含义和助记
3. 尝试记忆单词

**系统处理：**
1. 初始化记忆参数
2. 设定初始复习间隔（通常是 10分钟）
3. 记录首次学习时间

---

#### 步骤 2：首次复习

**用户操作：**
1. 在10分钟后收到复习提醒
2. 尝试回忆单词
3. 评分记忆表现（忘记/困难/一般/简单）

**评分标准：**
- **忘记**（Rating 1）：完全想不起来
- **困难**（Rating 2）：勉强想起来
- **一般**（Rating 3）：有些犹豫
- **简单**（Rating 4）：轻松回忆

**系统处理：**
根据评分更新记忆参数：
- **稳定性**：增加或减少
- **难度**：根据评分调整
- **检索率**：重置为 100%

**计算下次复习间隔：**
```
下次复习间隔 = 稳定性 × 系数
```

**示例：**
- 评分"简单"（4分）：稳定性 = 2天 → 下次复习间隔 = 2天
- 评分"一般"（3分）：稳定性 = 1天 → 下次复习间隔 = 1天
- 评分"困难"（2分）：稳定性 = 0.5天 → 下次复习间隔 = 12小时

---

#### 步骤 3：后续复习

**用户操作：**
1. 在指定的复习时间收到提醒
2. 尝试回忆单词
3. 评分记忆表现

**系统处理：**
1. 计算当前检索率
2. 根据评分更新记忆参数
3. 计算新的稳定性
4. 安排下次复习时间

**稳定性更新公式（简化版）：**
```
新的稳定性 = 旧稳定性 × (1 + 系数 × 评分)
```

**示例：**
- 当前稳定性 = 3天
- 评分"简单"（4分）：新的稳定性 = 3 × (1 + 0.5 × 4) = 9天
- 评分"一般"（3分）：新的稳定性 = 3 × (1 + 0.3 × 3) = 5.7天
- 评分"困难"（2分）：新的稳定性 = 3 × (1 + 0.1 × 2) = 3.6天

---

#### 步骤 4：长期巩固

**复习间隔逐渐延长：**
- 第1次复习：1天
- 第2次复习：3天
- 第3次复习：10天
- 第4次复习：30天
- 第5次复习：90天
- 第6次复习：180天
- ...

**记忆稳定性逐渐增强：**
- 初期：稳定性较低，需要频繁复习
- 中期：稳定性提升，复习间隔延长
- 后期：稳定性很高，复习间隔大幅延长

### 具体差异

#### 1. 间隔计算方式

**传统算法：**
```
下次复习间隔 = 上次间隔 × 固定倍数
```
- 固定倍数：通常是 2-3
- 示例：1天 → 3天 → 9天 → 27天 → 81天

**FSRS 算法：**
```
下次复习间隔 = f(稳定性, 难度, 评分)
```
- 考虑多个因素
- 示例：
  - 评分"简单"：1天 → 3天 → 10天 → 35天 → 120天
  - 评分"困难"：1天 → 1.5天 → 3天 → 6天 → 12天

---

#### 2. 个性化程度

**传统算法：**
- 所有人使用相同的间隔
- 所有单词使用相同的间隔
- 不考虑记忆表现差异

**FSRS 算法：**
- 每个人有不同的记忆参数
- 每个单词有不同的难度
- 根据记忆表现动态调整

**示例：**
- 用户A记忆好：1天 → 5天 → 20天 → 80天
- 用户B记忆差：1天 → 2天 → 5天 → 12天

---

#### 3. 复习效率

**传统算法：**
- 可能过早复习：浪费时间
- 可能过晚复习：容易遗忘
- 平均记忆保持率：60-70%

**FSRS 算法：**
- 精准把握最佳复习时机
- 避免过早或过晚复习
- 平均记忆保持率：80-90%

## 数学模型

### 核心公式

#### 1. 检索率计算

**公式：**
```
R(t) = e^(-t / S)
```

**变量说明：**
- `R(t)`：t时刻的检索率
- `t`：从上次复习到现在的时长
- `S`：稳定性
- `e`：自然对数的底（约等于 2.718）

**示例：**
- 稳定性 S = 10天
- t = 5天：R = e^(-5/10) = e^(-0.5) ≈ 60.7%
- t = 10天：R = e^(-10/10) = e^(-1) ≈ 36.8%
- t = 20天：R = e^(-20/10) = e^(-2) ≈ 13.5%

---

#### 2. 稳定性更新

**公式（简化版）：**
```
S_new = S_old × (1 + k × R)
```

**变量说明：**
- `S_new`：新的稳定性
- `S_old`：旧的稳定性
- `k`：学习系数（通常为 0.1-0.3）
- `R`：评分（1-4分）

**示例：**
- 当前稳定性 S_old = 5天
- 评分 R = 4（简单）
- 学习系数 k = 0.15
- 新稳定性 S_new = 5 × (1 + 0.15 × 4) = 8天

---

#### 3. 难度更新

**公式（简化版）：**
```
D_new = D_old + (R - 3) × α
```

**变量说明：**
- `D_new`：新的难度
- `D_old`：旧的难度
- `R`：评分（1-4分）
- `α`：难度调整系数（通常为 0.05-0.1）

**示例：**
- 当前难度 D_old = 5
- 评分 R = 2（困难）
- 难度调整系数 α = 0.08
- 新难度 D_new = 5 + (2 - 3) × 0.08 = 4.92

## 在应用中的实现

### 技术架构

**智能单词记忆应用中的实现：**

#### 1. 数据库设计

**单词记录表：**
```sql
CREATE TABLE words (
  id INTEGER PRIMARY KEY,
  word TEXT NOT NULL,
  meaning TEXT,
  stability REAL DEFAULT 1.0,      -- 稳定性
  difficulty REAL DEFAULT 5.0,     -- 难度
  next_review_date TEXT,           -- 下次复习日期
  last_review_date TEXT,           -- 上次复习日期
  review_count INTEGER DEFAULT 0,  -- 复习次数
  mastery_level REAL DEFAULT 0.0   -- 掌握率
);
```

#### 2. 复习流程

**步骤 1：获取待复习单词**
```javascript
// 查询所有需要复习的单词
const wordsToReview = await db.query(
  "SELECT * FROM words WHERE next_review_date <= ?",
  [currentDate]
);

// 按优先级排序（优先复习紧急的）
wordsToReview.sort((a, b) => a.next_review_date - b.next_review_date);
```

**步骤 2：用户评分**
```javascript
// 用户回忆后评分
function onReview(word, rating) {
  // 1. 计算当前检索率
  const retrievability = calculateRetrievability(
    word.stability,
    word.last_review_date
  );

  // 2. 更新记忆参数
  const newStability = calculateNewStability(
    word.stability,
    rating
  );

  const newDifficulty = calculateNewDifficulty(
    word.difficulty,
    rating
  );

  // 3. 计算下次复习间隔
  const nextReviewInterval = calculateNextInterval(
    newStability,
    newDifficulty
  );

  // 4. 更新数据库
  await db.update(word.id, {
    stability: newStability,
    difficulty: newDifficulty,
    last_review_date: currentDate,
    next_review_date: addDays(currentDate, nextReviewInterval),
    review_count: word.review_count + 1
  });
}
```

### 掌握率计算

**多维度评估：**

```javascript
function calculateMasteryLevel(word) {
  // 1. 记忆强度（基于稳定性）
  const memoryStrength = Math.min(word.stability / 30, 1.0);

  // 2. 复习频率（基于复习次数）
  const reviewFrequency = Math.min(word.review_count / 10, 1.0);

  // 3. 正确率（基于历史评分）
  const accuracy = calculateAverageRating(word) / 4;

  // 4. 时间跨度（从首次学习到现在）
  const timeSpan = Math.min(
    (currentDate - word.first_learned_date) / (90 * 24 * 60 * 60 * 1000),
    1.0
  );

  // 综合计算掌握率
  const masteryLevel =
    (memoryStrength * 0.3) +
    (reviewFrequency * 0.2) +
    (accuracy * 0.3) +
    (timeSpan * 0.2);

  return masteryLevel;
}
```

## 使用指南

### 评分标准

**FSRS 采用 4 级评分制：**

| 评分 | 名称 | 说明 | 稳定性变化 |
|------|------|------|-----------|
| 1 | **忘记** | 完全想不起来 | 大幅降低 |
| 2 | **困难** | 勉强想起来，花费时间 | 小幅降低或不变 |
| 3 | **一般** | 有些犹豫，但能回忆起来 | 适度增加 |
| 4 | **简单** | 轻松回忆，无需思考 | 大幅增加 |

**评分建议：**
- **诚实评分**：不要故意打高分
- **根据记忆质量**：不只看是否答对，还要看回忆的流畅度
- **考虑时间因素**：回忆时间超过 10 秒，降低评分

---

### 复习策略

#### 1. 每日复习

**建议时间：** 15-30 分钟

**复习数量：** 10-20 个单词

**最佳时间：**
- 早晨：精力充沛，记忆效率高
- 晚睡前：有助于长期记忆巩固

---

#### 2. 新单词学习

**建议数量：** 5-10 个/天

**学习流程：**
1. 查看单词拆分
2. 学习助记句子
3. 理解单词含义
4. 尝试记忆

**首次复习：**
- 时间：学习后 10 分钟
- 评分：通常为 "一般" 或 "简单"

---

#### 3. 间隔把握

**不要跳过复习：**
- 即使感觉记得很清楚，也要按时复习
- 系统安排的间隔是经过计算的

**不要提前复习：**
- 提前复习会降低算法的准确性
- 浪费时间和精力

## 常见问题

### Q1: 为什么我的复习间隔和别人不一样？

**A:** FSRS 是高度个性化的算法，每个人的记忆能力、学习习惯、单词掌握程度都不同，因此复习间隔也会不同。这是正常现象，也是算法的优势。

---

### Q2: 如果我忘记了一个单词怎么办？

**A:** 评分为 "忘记"（1分）后，系统会：
1. 重新安排该单词的复习间隔（通常较短）
2. 降低该单词的稳定性
3. 增加该单词的难度

不用担心，几次复习后就能重新掌握。

---

### Q3: 为什么有些单词的复习间隔很长？

**A:** 当一个单词的稳定性很高时（例如超过 30 天），说明你已经很好地掌握了它，系统会安排较长的复习间隔。这是正常的，也是高效的。

---

### Q4: 我可以手动调整复习间隔吗？

**A:** 不建议手动调整。FSRS 的间隔是经过科学计算的，手动调整会破坏算法的准确性，降低学习效率。

---

### Q5: FSRS 适合所有人吗？

**A:** FSRS 适合大多数学习者，但以下情况可能需要调整：
- 记忆能力极强或极弱的人
- 有特殊学习需求的人
- 学习目标非常明确的人

对于大多数学习者，FSRS 都是最佳选择。

---

### Q6: FSRS 算法会泄露我的学习数据吗？

**A:** 在智能单词记忆应用中，所有学习数据都存储在本地，不会上传到服务器。你的学习隐私完全受到保护。

---

### Q7: FSRS 的准确率有多高？

**A:** 根据大量用户数据统计，FSRS 的记忆保持率在 80-90% 之间，比传统算法高 15-20 个百分点。

---

### Q8: FSRS 和艾宾浩斯有什么区别？

**A:** 艾宾浩斯是一个记忆曲线理论，描述记忆随时间衰减的规律；FSRS 是一个具体的算法实现，基于艾宾浩斯理论，但加入了个性化调整和机器学习优化。
