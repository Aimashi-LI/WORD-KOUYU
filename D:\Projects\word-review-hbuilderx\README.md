# 单词记忆应用 - HBuilderX版本

这是使用 HBuilderX (uni-app) 框架重新开发的单词记忆应用，可以直接在 HBuilderX 中打包为 Android APK。

## 📱 应用功能

- ✅ **单词管理**: 添加、查看、删除单词
- ✅ **单词拆分**: 点击字母进行拆分记忆
- ✅ **助记句**: 添加个性化助记句子
- ✅ **智能复习**: 基于 FSRS 算法的间隔重复复习系统
- ✅ **掌握程度**: 多维度掌握率计算（低/中/高）
- ✅ **自动调度**: 智能优先级排序，确保紧急单词优先复习
- ✅ **数据统计**: 实时查看复习进度和掌握情况

## 🚀 使用 HBuilderX 打包 APK

### 步骤 1: 导入项目到 HBuilderX

1. 打开 HBuilderX
2. 选择 **文件** → **导入** → **从本地目录导入**
3. 选择 `D:\Projects\word-review-hbuilderx` 目录
4. 点击 **选择文件夹**

### 步骤 2: 安装依赖

在 HBuilderX 的终端中执行：

```bash
npm install
```

### 步骤 3: 配置应用签名

1. 在 HBuilderX 中，右键点击项目根目录
2. 选择 **发行** → **原生App-云打包**
3. 或者 **发行** → **原生App-本地打包**

#### 创建签名密钥

**方法 1：使用 HBuilderX 生成**

1. 选择 **发行** → **原生App-云打包**
2. 点击 **使用云端证书** 或 **使用自有证书**
3. 如果选择自有证书，填写信息：
   - **证书别名**: word-review
   - **证书密码**: (记住密码)
   - **私钥密码**: (记住密码)

**方法 2：手动生成签名文件**

```bash
# 使用 keytool 生成密钥库
keytool -genkey -v -keystore word-review.keystore -alias word-review -keyalg RSA -keysize 2048 -validity 10000
```

### 步骤 4: 配置打包选项

1. 选择 **发行** → **原生App-云打包** 或 **原生App-本地打包**
2. 填写以下信息：
   - **应用名称**: 单词记忆
   - **应用包名**: com.yourcompany.wordreview (建议修改)
   - **版本号**: 1.0.0
   - **版本名称**: 1.0.0
   - **证书**: 选择刚才创建的签名文件
   - **打包平台**: Android
   - **渠道**: 自定义 (可选)

### 步骤 5: 开始打包

1. 点击 **打包** 按钮
2. 等待打包完成
3. 打包成功后，APK 文件会下载到本地

### 步骤 6: 安装 APK

**方法 1：通过 USB 安装**

```bash
# 连接手机后执行
adb install word-review.apk
```

**方法 2：直接传到手机安装**

1. 通过 USB、微信、QQ 等方式将 APK 传到手机
2. 在手机上打开 APK 文件
3. 允许安装未知来源应用
4. 点击安装

## 🎯 核心功能说明

### 1. FSRS 智能复习算法

使用 Free Spaced Repetition Scheduler (FSRS) 算法，实现个性化的间隔重复复习：

- **初始期（0-1天）**: 快速增长
- **成长期（1-7天）**: 稳步提升
- **稳定期（7-30天）**: 平缓增长
- **巩固期（30天+）**: 缓慢巩固

### 2. 掌握程度计算

基于稳定性和总体掌握率的综合判断：

- **低掌握**: 稳定性 < 2天
- **中掌握**: 稳定性 2-10天
- **高掌握**: 稳定性 > 10天

### 3. 智能复习队列

- 按紧急程度排序
- 优先复习超期单词
- 自动补充机制
- 队列大小恒定（20个）

## 🔧 技术栈

- **框架**: HBuilderX + uni-app
- **语言**: Vue 3
- **数据库**: SQLite (本地)
- **算法**: FSRS (间隔重复)

## 📝 注意事项

### 应用签名

- **妥善保管签名文件** (`.keystore` 或 `.jks`)
- **记住所有密码**（证书密码 + 私钥密码）
- **备份签名文件**到安全的地方
- **丢失签名文件 = 无法更新应用**

### 应用包名

建议修改为有意义的包名：

编辑 `manifest.json`：

```json
{
  "appid": "com.yourcompany.wordreview"
}
```

编辑 `pages.json` 中的应用名称。

### 网络依赖

HBuilderX 云打包需要网络连接，如果网络不稳定：
- 可以使用本地打包
- 或者使用其他打包工具

## 🆚 与 React Native 版本的区别

| 特性 | React Native 版本 | HBuilderX 版本 |
|------|------------------|----------------|
| **开发工具** | VS Code + Android Studio | HBuilderX |
| **打包方式** | Gradle 命令行 | HBuilderX 云/本地打包 |
| **网络要求** | 需要稳定的 Gradle 依赖下载 | 需要网络（云打包）|
| **学习曲线** | 较陡峭 | 较平缓 |
| **开发效率** | 中等 | 较高 |
| **调试体验** | 需要 React Native 环境 | HBuilderX 内置调试 |

## 📞 常见问题

### Q: 打包失败怎么办？

A: 检查以下几点：
1. 确认签名文件配置正确
2. 检查应用包名是否符合规范
3. 确认网络连接正常
4. 查看 HBuilderX 控制台的错误信息

### Q: 如何修改应用图标？

A: 将应用图标放在 `static/` 目录下，然后在 `manifest.json` 中配置。

### Q: 如何添加更多功能？

A: 可以在 `pages/` 目录下创建新页面，并在 `pages.json` 中注册。
